FAQ

Security, AI, and workflow questions.

Is my CSV uploaded when I use the public audit tool?

No. The public tool runs as static JavaScript in your browser and parses the selected CSV locally. CRM Import Rescue does not receive that file unless you choose to send a sample or paid cleanup file separately.

Do you use AI to clean customer data?

The browser tool uses deterministic checks and cleanup rules, not an AI model. Paid cleanup is handled with explicit rules, scripts, and review. Raw customer CSVs are not pasted into AI tools unless you explicitly approve a narrow, anonymized use.

Will my data be used to train AI models?

No. CRM Import Rescue does not use customer CSVs to train models, create examples, or build marketing samples. Public examples use synthetic sample data.

Do you need access to my CRM?

No. The normal workflow uses exported CSV files only. You keep control of HubSpot, Salesforce, Airtable, and any other CRM account.

What should I send first?

Start with 20-100 anonymized rows that preserve the same column names and field patterns as the real file. Remove or replace sensitive names, emails, phone numbers, notes, and IDs until scope and handling are agreed.

What if the file contains sensitive or regulated data?

Do not send sensitive, regulated, or contract-restricted data through the intake form. Start with anonymized structure only. If the real file requires special handling, that needs to be agreed before any full-file transfer.

What happens to working files after delivery?

File handling and deletion expectations should be agreed before a full cleanup. The default goal is narrow: use the file to produce the cleaned CSV, issue export, and blocker report, not to build a retained data asset.

What does this do that Excel does not?

Excel is useful for quick edits. CRM Import Rescue is designed to produce a repeatable cleanup path: cleaned import CSV, row-level issue CSV, blocker report, and rules that can be reused for the next event list, export, or migration.

What does the $500+ reusable workflow include?

The $500+ workflow is a one-time setup for one recurring CSV format, one CRM target, and one agreed rule set. It includes the reusable cleanup script or workflow, a cleaned-output template, an issue-report template, a handoff walkthrough, and 14 days of small fixes after delivery.

How long does the $500+ workflow last?

It lasts as long as the source file structure and cleanup rules stay the same. If the vendor changes columns, the CRM mapping changes, or new cleanup rules are needed, that is a new paid change request or a managed monthly cleanup engagement.

How is reusable workflow scope tracked by the code?

Each reusable workflow has a versioned JSON contract with expected columns, required columns, CRM target, cleanup rules, and included-fix window. The tool compares each new CSV against that contract before export and flags changed schemas as matched, blocked, or needing a paid update.

Can an agency use this for client work?

Yes. The service is a good fit for CRM consultants, RevOps teams, and agencies that need a small subcontracted cleanup sprint before a client import or migration.
